# Acceptance Criteria V1

Last updated: 2026-03-18

## Dashboard

- Shows current month summary cards.
- Shows recent transactions.
- Shows category spending chart.

## Transactions

- User can add, edit, and delete transactions.
- Filters work correctly.
- Account balances update after transaction changes.

## Budgets

- User can define monthly budgets.
- Budget progress reflects actual spend.
- Over-budget states are visible.

## Goals

- User can create goals and contribute to them.
- Goal progress percentage updates correctly.

## Reports

- Date filters update charts and summary data.
- Export returns the correct filtered dataset.

## Usage Note

- These acceptance criteria should be used as the functional definition of done for the corresponding V1 modules.
- Detailed test cases can be derived from these criteria later during implementation and QA planning.
